接口隔离（Interface Segregation Principle）（降低依赖，耦合。 一定要适度，接口设计的过大或过小都不好）
===================================================================
> 使用多个隔离的接口，比使用单个接口要好。还是一个降低类之间的耦合度的意思
接口尽量小，但是要有限度。但是如果过小，则会造成接口数量过多，使设计复杂化。所以一定要适度
